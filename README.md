# Frontier News

Static SEO landing site for **Frontier News by Hyperjump** — curated tech news and AI summaries from trusted YouTube channels.

Live URL target: [https://frontiernews.tech/](https://frontiernews.tech/)

## Stack

- Static HTML generated by `build.mjs` (Node 18+)
- No runtime framework — server-rendered markup for SEO and Core Web Vitals
- Locales: English (default), Indonesian, German, Italian, French, Arabic (RTL), Japanese, Sundanese

## Develop

```bash
npm run build
npm run dev
```

`npm run build` writes the site to `dist/`. `npm run dev` builds and serves `dist/` on port 4173.

## Deploy to GitHub Pages

1. Push this repo to GitHub.
2. Settings → Pages → Source: **GitHub Actions**.
3. The workflow in `.github/workflows/deploy-pages.yml` builds and deploys `dist/` on pushes to `main`/`master`.
4. Optional custom domain: `dist/CNAME` is set to `frontiernews.tech`. Point DNS accordingly.

You can also publish `dist/` manually to the `gh-pages` branch if you prefer.

## Content

| Path                    | Purpose                   |
| ----------------------- | ------------------------- |
| `content/articles.json` | Digest/summary records    |
| `content/topics.json`   | Topic archive definitions |
| `content/i18n/*.json`   | UI copy per language      |
| `public/assets/`        | CSS, JS, static assets    |

After editing content, run `npm run build`.

## Newsletter / Cloudflare Turnstile

The subscribe form posts to the backend and requires a Cloudflare Turnstile token, matching the Hyperjump site form.

Set these before building:

```bash
cp .env.example .env
# fill NEXT_PUBLIC_TURNSTILE_SITE_KEY
export $(grep -v '^#' .env | xargs)
npm run build
```

For GitHub Pages, add repository secret `NEXT_PUBLIC_TURNSTILE_SITE_KEY` (and optional variable `NEXT_PUBLIC_FRONTIERNOTES_API_URL`).
